## 1. 분야 개요

금융 분야는 공공 API 중 가장 많은 21개 독립 포털을 운영하며, 기관별 파편화가 가장 심한 분야이다. DART(금감원), ECOS(한국은행), KRX(한국거래소) 등 핵심 데이터가 각각 별도 포털에서 독립 운영되고, data.go.kr과의 연계가 미비하다.

## 3. 분야별 특성

### 3.1 강점
- **커뮤니티 생태계가 가장 활발**: pykrx(954 stars), FinanceDataReader(1,400 stars), dart-fss(360 stars), OpenDartReader(433 stars)
- **DART**: 공공 API 중 유일하게 다수의 독립 SDK/라이브러리 보유
- **금융결제원**: 공공 API 중 유일하게 OAuth 2.0 + JSON 전용 조합 (AI 등급 C로 최상위 그룹)

### 3.2 약점
- **21개 포털 파편화**: 금융 데이터를 종합적으로 활용하려면 10개 이상의 별도 ID 필요
- **ECOS의 독특한 Path Param 방식**: `/api/키/JSON/서비스코드/시작/끝/항목1/항목2/...` 형태로, 순서 의존적이라 프로그래매틱 호출이 어려움
- **KRX 접근 장벽**: 본인인증 + 법인사업자 등록 요구. 개인 개발자 접근 사실상 차단

### 3.3 AI 적합성
- 금융 21개 포털 중 AI 등급 C 이상: **1개** (금융결제원, C등급)
- 나머지 20개: D등급 14개, E등급 6개
- **마이데이터 규격(OAuth 2.0 기반)**이 유일한 AI-Ready 아키텍처이나, 개인정보 동의 절차가 복잡

## 5. 제언

1. **DART + ECOS + KRX 통합 API 게이트웨이** 구축 — 금융 데이터의 원스톱 접근
2. **ECOS Path Param 방식 → REST Query Param 방식** 전환 권장
3. **KRX 가입 장벽 완화** — 개인 개발자도 기본 시세 데이터 접근 가능하도록
4. 금융결제원 오픈뱅킹의 OAuth 2.0 모델을 **금융 분야 표준 인증 패턴**으로 확산

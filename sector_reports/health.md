# 건강/보건의료 분야 공공 API 현황 보고서

**분야 코드**: HLTH (건강/보건의료)
**작성일**: 2026-04-02
**대상 포털 수**: 8개

---

## 1. 분야 개요

보건의료 분야는 8개 포털이 건강보험, 질병관리, 의약품 안전, 건강검진 등을 담당한다. 개인 건강정보의 민감성으로 인해 접근 장벽이 상대적으로 높으며, 마이데이터(마이헬스웨이) 정책이 분야 전체의 방향성을 바꾸고 있다.

## 2. 포털 현황

| 포털 | 편의성 등급 | AI 등급 | 인증 | 기본포맷 | 특징 |

| 건강보험심사평가원 | D (5) | D (3) | API Key Query | XML | 병원/약국 정보, 진료비 |

## 3. 분야별 특성

### 3.1 강점
- **마이헬스웨이의 혁신적 아키텍처**: AI 등급 B(9점)로 보건 분야 유일의 AI-Ready 포털. OAuth 2.0, JSON 전용, 표준 HTTP 상태코드 사용
- **데이터 중요성**: 건강보험 데이터는 전 국민 커버리지(98%+)로 연구 가치 극히 높음
- **질병관리청**: 코로나19 이후 감염병 데이터 개방이 확대됨

### 3.2 약점
- **마이헬스웨이 외 전 포털 E등급**: 7개 포털이 AI 등급 E(2점)로 최하위
- **SDK/MCP 전무**: 8개 포털 모두 SDK, MCP 서버 없음
- **건강보험공단 접근 장벽**: 개인 건강데이터 조회 시 공동인증서 또는 마이데이터 동의 필요
- **XML 중심**: 마이헬스웨이를 제외한 7개 포털이 XML 기본

### 3.3 AI 적합성 — 마이헬스웨이의 이중적 위치
- 마이헬스웨이는 기술적으로 AI-Ready(B등급)이지만, **개인정보 동의 절차가 복잡**하여 AI Agent의 자율적 데이터 수집에는 제약
- 익명화된 통계 데이터(심평원, 질병관리청)는 AI 활용 가능하나 XML/비표준 에러로 자동화 어려움
- **HL7 FHIR 표준** 채택이 국제적 흐름이나, 한국 공공 보건 API는 아직 미채택

마이데이터 통합 이관으로 폐기. 4개월 사전 공지로 비교적 체계적 전환.

## 5. 제언

1. **마이헬스웨이 아키텍처를 보건 분야 표준으로 확산** — OAuth 2.0 + JSON + 표준 HTTP 코드
2. **HL7 FHIR 표준 채택 로드맵** 수립 — 국제 의료 데이터 상호운용성 확보
3. **익명화 통계 API의 AI 등급 개선** — 심평원, 질병관리청 데이터는 개인정보 이슈 없이 JSON/OpenAPI로 전환 가능
4. **보건 데이터 MCP 서버** 개발 — 감염병 모니터링, 의약품 검색 등 AI 활용 잠재력 높음
5. **건강보험 데이터의 연구용 API** 별도 제공 — 전수 데이터 기반 의료 AI 연구 촉진

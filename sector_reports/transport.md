**작성일**: 2026-04-02
**대상 포털 수**: 12개

## 1. 분야 개요

교통/물류 분야는 12개 포털이 도로, 철도, 항공, 해운, 대중교통 등 영역별로 분산 운영된다. 실시간 교통정보(ITS), 대중교통 시간표(TAGO), 고속도로 정보(한국도로공사) 등 일상 밀착형 데이터를 제공하지만, 포털 간 데이터 연계가 미흡하다.

## 3. 분야별 특성

### 3.1 강점
- **실시간 데이터 풍부**: ITS 교통소통, TAGO 도착정보, 도로공사 실시간 정보 등
- **data.go.kr 연계율 상대적 양호**: 대부분 data.go.kr 경유 접근 가능
- **일상 활용 빈도 높음**: 교통 앱, 내비게이션, 물류 시스템에서 활발히 사용

### 3.2 약점
- **전 포털 AI 등급 D~E**: C 등급 이상이 0개. AI Agent가 자율적으로 교통 데이터를 활용하는 것이 사실상 불가
- **SDK/MCP 전무**: 12개 포털 모두 SDK, MCP 서버 없음
- **XML 중심**: 12개 중 10개가 XML 기본. JSON 전용은 해양수산빅데이터 1개뿐
- **실시간 데이터 품질 불투명**: "실시간" 표기해도 실제 갱신 주기가 수분~수시간 지연되는 사례 존재

### 3.3 AI 적합성
- 12개 포털 모두 AI 등급 D 또는 E
- OpenAPI 스펙 미제공, HTTP 상태코드 미사용, CORS 미지원
- 교통 분야는 실시간성이 핵심인데, Rate Limit 헤더 미제공으로 호출 관리 불가

## 5. 제언

1. **교통 통합 API 게이트웨이** 구축 — ITS + TAGO + 도로공사 + KRIC을 단일 진입점으로
2. **실시간 데이터에 WebSocket/SSE 옵션** 추가 — 폴링 방식의 한계 극복
3. **GTFS(General Transit Feed Specification) 표준 채택** — 국제 표준으로 교통 데이터 상호운용성 확보
4. **교통 데이터 MCP 서버** 우선 개발 — 내비게이션 AI Agent 활용 잠재력 높음
